# A-Machine 스마트폰 전용 UI 전면 개편 구현 계획

현재 'A-Machine' 프로젝트의 프론트엔드를 **스마트폰 전용 UI 중심으로 개편**합니다. 대시보드를 완전히 제거하고, `PhoneUI`를 화면 중앙에 배치하며, 대기 상태와 통화 상태에 따라 리얼한 폰 화면을 보여주도록 개선합니다.

---

## User Review Required

> [!IMPORTANT]
> **1. 대시보드 제거 및 PhoneUI 전면 배치**
> - 기존의 2단 데스크톱 그리드 시스템(`.main-grid`)과 좌측 `Dashboard` 패널을 완전히 삭제합니다.
> - 대신 화면 중앙에 아름답고 입체적인 **3D 스마트폰 쉘(Phone Shell)**을 배치하여 모바일 기기 화면을 단독으로 주시할 수 있도록 설계합니다.
>
> **2. [스피커폰] 버튼과 에코 방지 모드(Echo Guard) 연동**
> - 하단 제어 버튼 중 **[스피커폰]** 버튼을 에코 방지 모드(`echoGuard`) 상태와 1:1로 직접 매핑합니다.
>   - 스피커폰 ON (활성화) = 에코 방지 모드 ON (스피커 출력이 마이크로 유입되는 현상 차단)
>   - 스피커폰 OFF (비활성화) = 에코 방지 모드 OFF (자연스러운 동시 대화 모드)

### 2. 컴포넌트 디자인 고도화

#### [MODIFY] [PhoneUI.jsx](file:///c:/Users/SKTelecom/skt/A-Machine_Workspace/client/src/components/PhoneUI.jsx)

- **상태에 따른 스크린 화면 분기**:
  1. **Idle/Ready (대기 화면)**:
     - 실제 스마트폰처럼 상단에 통신사, 안테나, 배터리 잔량, 시간이 들어간 **Status Bar**를 렌더링합니다.
     - iOS 스타일의 고품격 **'최근 통화 기록(Recents)'** 화면을 기본 인터페이스로 제공합니다.
       - "오늘", "어제" 구분선 제공.
       - `아빠 💖`, `김 부장님`, `이모` 등의 가상 인물 사이에 **`A-Machine 통화 비서`**(부재중 표시 - 빨간색 번호 강조)가 포함됩니다.
       - 리스트 아이템을 클릭하거나, 리스트 우하단에 배치된 **둥근 녹색 플로팅 통화 버튼(FAB)**을 누르면 즉시 통화 연결(`onStartCall()`)이 시작됩니다.
       - 하단에 **[즐겨찾기, 최근 통화, 연락처, 키패드]**로 이루어진 미니 탭 바를 배치해 완벽한 전화 앱 뷰를 모사합니다.
     
  2. **Connecting (연결 중 화면)**:
     - 폰 스크린 전체가 서서히 에이닷 풍의 화려한 그라데이션으로 전환됩니다.
     - 중앙에 입체적인 아바타와 "연결 중..." 타이핑 웨이브 인디케이터가 부드럽게 렌더링됩니다.

  3. **Active (통화 중 화면 - 에이닷 스타일)**:
     - 화려한 유체 그라데이션 광원 배경 위에 투명한 **글래스모피즘(Glassmorphism) 카드** 형태로 대화 기록 창이 흐르게 배치됩니다.
     - 중앙의 아바타는 광원 아우라(Aura)를 입고 둥실둥실 떠오르며, 특히 **에이전트가 답변을 말하고 있는 상태(`isModelSpeaking === true`)일 때 아바타의 외부 네온 링이 역동적으로 호흡(Pulse)하는 미크로 애니메이션**을 보여줍니다.
     - 대화창의 스피커 버블은 매우 정제된 형태의 유리 버블로 디자인되어 가독성을 높입니다.
     - 하단에는 에이닷의 정체성을 담은 **4버튼 패널 [음소거] [키패드] [스피커폰] [종료]**을 구현합니다.
       - [스피커폰] 버튼은 `echoGuard`가 ON일 때 아름다운 청록색 테두리로 활성화 상태를 표현합니다.
       - [종료] 버튼은 전용 레드 컬러의 서클 버튼으로 매칭되어 직관적으로 통화를 끊을 수 있게 합니다.

## Verification Plan

### Automated Tests & Rendering Verification
- `client` 폴더 내부의 Vite 빌드를 실행하여 문법 오류나 종속성 누락이 없는지 검증합니다.
- CSS 스타일 충돌이나 미디어 쿼리 버그가 없도록 브라우저 테스트 툴을 활용하여 다양한 크기(데스크톱 및 모바일)에서 폰 프레임이 완벽히 중앙 정렬되는지 확인합니다.

### Manual Verification
- 가상의 최근 통화 이력 리스트에서 전화를 눌렀을 때 실제로 통화 상태(`connecting` -> `active`)로 매끄럽게 넘어가는지 확인합니다.
- AI의 발화 상태 변화에 따라 중앙 아우라 링이 실시간으로 활성화되며 부드럽게 진동하는지 모니터링합니다.
- 스피커폰 버튼 클릭 시 에코 방지 상태가 정상적으로 ON/OFF 전환되는지 콘솔 및 동작을 테스트합니다.
