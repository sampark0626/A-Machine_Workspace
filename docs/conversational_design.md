# 🎙️ A-Machine Conversational Speech Design Guide
> 에이머신(A-Machine)의 한국어 발화 자연스러움 극대화를 위한 기술적/대화적 설계 기록

본 문서는 에이머신 프로젝트가 OpenAI Realtime API 기반의 대화형 자동 응답기(Answering Machine)로서, **발신자의 이질감을 원천 차단하고 통화 지속성을 극대화하기 위해 수행한 모든 기술적 및 대화 설계적 노력**을 상세히 기록한 마스터 피스 문서입니다. 해커톤 발표 자료(PPT, 데모 시나리오, 기술 명세) 작성 시 핵심 자료로 즉각 활용할 수 있도록 정밀하게 구성되었습니다.

---

## 1. 🎯 발화 자연스러움(Speech Naturalness)의 목적 및 배경

### 1) 사용자 경험(UX) 관점의 페인 포인트 (Pain Points)
* **기계적인 자동 응답에 대한 심리적 거부감**: 일반적인 발신자는 사람이 전화를 받지 않아 기계(ARS 또는 단순 음성사서함)로 전환되는 순간 **대화를 시도하지 않고 즉시 전화를 끊는(Call Abandonment) 성향**이 매우 강합니다.
* **이질감으로 인한 대화 단절**: 음성이 획일적인 로봇 톤이거나, 문맥에 맞지 않는 "알겠습니다"를 연발하는 순간 사용자는 자신이 무의미한 기계와 대화하고 있다고 인지하여 용건을 남기거나 일정을 잡으려는 시도를 포기합니다.

### 2) 에이머신의 최종 목표 (Answering Machine KPI)
* **"인간 비서와의 통화와 구별할 수 없는 수준의 완벽한 핑퐁 대화"**
* 발신자가 전화를 걸었을 때 AI가 대리 수신했음을 인지하더라도, 인간 수준의 매끄러운 호흡과 리액션, 스마트한 일정 확인 기능에 몰입하여 **통화를 끊지 않고 목적(용건 전달 또는 일정 조율)을 100% 완벽하게 달성하도록 유도**합니다.

---

## 2. 🛠️ 기술적 개선 및 오디오 스트리밍 튜닝 (Technical Implementations)

### 1) End-to-End 실시간 음성-음성(Speech-to-Speech) 아키텍처 전환
* 기존의 **STT(음성 인식) ➡️ LLM(텍스트 추론) ➡️ TTS(오디오 합성)**로 이어지는 직렬 파이프라인 방식을 탈피했습니다.
* **OpenAI Realtime API (`gpt-realtime-2`)** 모델을 활용해 오디오 신호를 텍스트 변환 과정 없이 다이렉트로 인코딩/디코딩하여 음향적 뉘앙스(톤, 감정, 호흡)를 즉각 처리합니다.
* 이를 통해 대화 지연 시간(Latency)을 **1초 이내**로 획기적으로 단축하여 대화의 맥이 끊기는 현상을 원천적으로 해결했습니다.

### 2) 자연스러운 VAD(음성 활성 감지) 및 시맨틱 턴 제어 (`semantic_vad`)
단순한 음압(볼륨) 기반의 음성 단절 감지는 사용자가 말을 하다가 1초 정도 생각하는 도중에 AI가 말을 치고 들어오는 최악의 이질감(Interruption)을 초래합니다.
* **시맨틱 VAD 모드 전격 도입**: `turn_detection` 속성을 `type: "semantic_vad"`로 설정하고 `eagerness: "auto"` 값을 적용했습니다.
* **효과**: AI가 단순히 사용자의 목소리가 안 들리는 침묵 시간만을 재는 것이 아니라, **화자의 문장이 문법적/meaning-wise로 마침표를 찍었는지 실시간 분석**합니다. 문장 중간의 호흡 쉼표에서는 말을 끊지 않고 기다려주며, 문장이 끝난 시점에는 칼같이 응답을 시작합니다.

### 3) 최적의 한국어 자연스러움을 탑재한 `Marin` 음색 도입
* OpenAI의 오디오 라인업 중 한국어 구현 및 연출력에서 가장 압도적인 자연스러움을 평가받은 최신 음색 **`Marin`**을 발굴하여 기본 시스템 음성으로 채택했습니다.
* `Marin`은 한국어 고유의 종결 어미 억양 처리와 실제 목소리의 공기 반 소리 반(Breathiness) 느낌을 훌륭하게 연출합니다.
* **비즈니스 연속성**: 통화 전에 사용자가 목소리를 직접 선택하여 체험할 수 있는 UI 칩을 프론트엔드에 추가하여 유연한 테스트 환경을 제공합니다.

## 3. 📝 대화 설계 및 프롬프트 엔지니어링 (Conversational & Prompt Engineering)

마치 실제 베테랑 비서가 생각하고 말하는 듯한 역동적인 언어 감각을 연출하기 위해, 시스템 프롬프트 및 메시지 생성 정책에 특수 설계 기법을 적용했습니다.

### 1) 마이크로 포즈(Micro-Pause) & 호흡 공학 설계 (`...` 및 `,` 배치)
AI 모델이 발화할 텍스트에 문장 부호를 전략적으로 배치하여 TTS 모델의 억양과 포즈를 세밀하게 조율합니다.

| 문장 부호 | 실제 작동 및 TTS 번역 효과 | 예시 문장 |
| :---: | :--- | :--- |
| **말줄임표 (`...`)** | 0.3~0.5초의 미세한 침묵(Micro-Pause)을 삽입하여 생각을 고르는 인간다운 자연스러움을 부여. 목소리의 끝맺음을 부드럽게 감속(Deceleration)함. | `"아, 잠시만요... 그날 일정을 한번 확인해 볼게요..."` |
| **쉼표 (`,`)** | 인위적으로 문장의 나열 사이에 상냥하게 반 호흡 쉬어가도록 유도하여 Flat한 로봇 톤을 제거. 쉼표 앞 단어의 피치(Pitch)를 살짝 올려 기대감을 형성. | `"대신 전화를 받은, AI 비서 에이머신입니다."` |

### 2) 상황 맥락을 완벽히 제시하는 오프닝 첫 인사 (Greeting Design)
기존의 뜬금없는 "용건이 무엇인가요?" 식의 첫인상을 버리고, 전화 부재 수신 상황을 정확히 정립하는 **Option A** 템플릿을 완성했습니다.

* **최종 오프닝 템플릿**:
  > `"안녕하세요! ... 지금 수민님이 부재중이시라... 대신 전화를 받은 AI 비서 에이머신입니다. 메모나 일정 등록이 가능한데, ... , 어떤 일로 전화주셨나요?"`
* **설계 포인트**:
  * `안녕하세요! ...` ➡️ 전화를 받자마자 바로 말을 쏟아내지 않고 0.3초의 정적을 주어 상대가 들을 준비를 할 수 있게 배려합니다.
  * `지금 수민님이 부재중이시라...` ➡️ 발신자가 전화를 건 대상(수민)이 부재중임을 합리적으로 인지시켜 혼란을 방지합니다.
  * `대신 전화를 받은 AI 비서 에이머신입니다.` ➡️ 본인의 정체성을 명확히 밝힙니다.
  * `메모나 일정 등록이 가능한데, ... , 어떤 일로...` ➡️ 메모와 캘린더 일정이 가능하다는 서비스 능력을 슬림하고 매끄럽게 고지하며, 발신자가 장황함을 느끼기 전에 핵심 타이밍에 신속히 답변할 수 있도록 마이크를 넘깁니다.

### 3) 🚫 중복 및 상투적 발화("알겠습니다" 앵무새 오류)의 차단 규정
통화 시 AI가 기계처럼 보이는 가장 큰 이유는 단어의 반복입니다. 특히 데이터 조회나 도구 실행 전후로 상투적인 "알겠습니다"를 연이어 말하는 패턴을 완벽히 방지하는 강력한 가이드라인을 수립했습니다.

* **동조어 연속 금지 정책 (Zero Consecutive Duplicate)**:
  * 직전 턴에 리액션으로 "알겠습니다"를 썼다면 바로 다음 턴에는 어떠한 경우에도 "알겠습니다"로 발화를 시작할 수 없습니다.
* **다채로운 리액션 풀(Filler Pool) 확보**:
  * 단순 수긍을 대체할 감성적이고 입체적인 리액션 단어들을 프롬프트에 정의하여 자연스럽게 번갈아가며 호출하도록 유도합니다.
  * *예시 리액션*: `"아~"`, `"음..."`, `"그렇군요!"`, `"네, 말씀하신 대로..."`, `"확인해 보니까..."`, `"아하!", "바로 조치해 드릴게요."`
* **도구 호출(Tool Call) 전후 호응 중복 제거**:
  * 기존 오류: *(도구 호출 전)* "알겠습니다. 조회할게요." ➡️ *(도구 호출 후)* "네, 알겠습니다. 조회해보니..." **(기계적인 상투어 중복 발생)**
  * 개선 방향: *(도구 호출 전)* "아, 잠시만요... 그날 캘린더를 확인해 볼게요." ➡️ *(도구 호출 후)* "음... 조회해 보니, 오후 2시부터 4시까지 비어 있네요." **(중복 없이 바로 본론으로 진입하는 입체적 흐름)**

## 4. 📈 해커톤 심사위원을 사로잡을 핵심 포인트 (Hackathon Winning Strategy)

1. **사용자 경험의 완전함 (Seamless UX)**:
   단순 장난감 수준의 챗봇이 아닌, **실제 수신자(수민)의 부재중 상황을 리얼하게 방어하고 캘린더 연동 및 통화 요약 메모 전달까지 원스톱으로 해결하는 완성도 높은 가상 비서 시나리오**가 준비되어 있습니다.
2. **청각적 놀라움 (Sonic Wow Factor)**:
   시연자가 첫 전화를 연결하는 순간 흘러나오는 `Marin` 보이스의 미세한 숨소리와 추임새(`음...`, `아, 잠시만요...`) 연출은 타 팀의 일반적인 텍스트 TTS 엔진이나 기계적인 기성 대화 엔진들과 차원이 다른 프리미엄 감성을 느끼게 합니다.
3. **지연 시간 파괴 (Sub-second Latency)**:
   인간이 대화 중 답답함을 느끼지 않는 골든타임인 "1초" 이내의 반응 속도로 핑퐁 대화를 이어갑니다.
